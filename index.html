HTML

  • The head element contains metadata to help the browser render the page properly
  • The body element contains content that is shown to the user
  • The HTML element is the opening tag, and represents the start of the HTML document.
  • The title element defines the title of the webpage (shown in the browser tab) . Important for SEO.
  • The style element defines the CSS styles associated with a webpage
  • The base element defines the base URL for a webpage
  • The link element connets an external resource to the HTML document
  • The meta element defines metadata such as character set, description, keywords, etc.
  • The script element embeds client-side scripts in the HTML document
  • Use href to specify the location of an external resource. This uses a relative path, and the location is derived from the current location.

CSS

  • Inline CSS are applied directly in the HTML file. They apply specifically to one HTML tag. E.g. attributes within an element
  • Internal CSS style sheets contain rules for the webpage in the [head] section of the HTML file. When done like this, they only apply to that page and can contain classes and IDs to identify elements
  • External CSS style sheets are the most common and are a separate file that contains all the rules. These are linked to the file using [link] within [head]
  • A selector defines the element or atatributes to which the declarations will apply
  • A declaration is made up of a property and a value.
  • You can combine two selectors within a single CSS rule with a comma. Declarations within will affect both/all.
  • A margin indicates how much space we want around the outside of the element and are not visible/
  • Padding is different to margin in that it adds space around the content inside an element. Larger padding means larger element as it is inside an element.

Git

Initialization

  • git init: initialize an existing directory as a git repository
  • git clone [url]: retrieve an entire repository from a hosted location via URL

Stage & Snapshot

  • git status: show modified files in working directory, staged for next commit
  • git add [file]: add a file as it looks now to next commit
  • git reset [file]: unstage a file while retaining changes in working directory
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not committed
  • git commit -m "message": commit staged content as a new commit snapshot

Branch & Merge

  • git branch: list your branches. an asterisk will appear next to the currently active branch
  • git -b [branch name]: create a new branch at the current commit
  • git checkout: switch to another branch and check it out into your working directory
  • git merge [branch]: merge the speicified branch's history into the current one
  • git log: show all commits in the current branch's history

Share & Update

  • git remote add [alias] [url]: add git URL as an alias
  • git fetch [alias]: fetch down all branches from that Git remote
  • git merge [alias]/[branch]: merge a remote branch into your current brance to bring it up to date
  • git push [alias] [branch]: transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch

Git Version Control


Make sure you're not working on main and have created a new branch. Check with git status.
Any unsaved changes will appear after git status.


  • -add: adds mods in the current branch to the staging area
  • -A or --all: indicates add all changes
  • git commit -m "message here": create snapshop of current changes with the comment

Before pushing to repo, check that the local branch is in sync with the base branch with: git pull origin main


  • -pull: receives a branch's mods into the local environment
  • -origin: indicates source of the pull will be the GitHub repo
  • main: indicates the branch in this instance

Therefore: git pull origin main pulls modifications from the main branch in the GitHub repo to local environment.


Once synced, push changes with git push origin (branch)


  • -push: pushes changes made locally
  • -origin: indicates destination of push (origin indicates github repo)

Then check GitHub for changes. From here we can create PRs and merge into main. Make sure in the PR that the arrow goes from branch to main.


JavaScript

  • A variable is a named container that allows us to store data in our code. Declare a variable by using the let or var keywords, followed by the name of the variable. Variables are case-sensitive.
  • Control flow is the order in which a computer executes code in a script.
  • Comments are snippets of text added alongside code. Usually to indicate something to person working/reading the code.
  • Operators are mathematical symbols that produce a result based on two values (or variables).
  • Conditionals are code structures used to test if an expression returns true or not. E.g. if(). The expression inside the parentheses is the test.
  • Functions are a way of packaging functionality that is to be reused. You can define a body of code as a function that executes when you call the function name in the code. It is a good alternative to repeatedly writing the same code. Anything that looks like a vairable name, but is followed by parentheses, is likely a function. Functions take 'arguments' which is the data that they need to run and can be separated by commmas if there is more than 1 argument.
  • Events are code structures that listen for activity in the browser, and run code in response.
